## What is NLP?

**Natural Language Processing (NLP)** is a branch of Artificial Intelligence that enables computers to *understand*, *interpret*, and *generate* human language.

NLP sits at the intersection of:
- Linguistics
- Computer Science
- Machine Learning

> "Language is the road map of a culture." — Rita Mae Brown

---

## Core NLP Tasks

Here are some of the most common tasks in NLP:

| Task | Description | Example |
|------|-------------|---------|
| **Tokenization** | Splitting text into words or sentences | `"Hello world"` → `["Hello", "world"]` |
| **POS Tagging** | Labeling parts of speech | `"runs"` → `VERB` |
| **NER** | Named Entity Recognition | `"Lahore"` → `LOCATION` |
| **Sentiment Analysis** | Detecting emotion or opinion | `"Great product!"` → `POSITIVE` |
| **Machine Translation** | Translating between languages | English → Urdu |
| **Text Summarization** | Condensing long documents | Article → 3-sentence summary |

## What is RAG?

**Retrieval-Augmented Generation (RAG)** is a technique that combines two powerful ideas:

1. **Retrieval** — searching a knowledge base for relevant documents
2. **Generation** — using a language model (like GPT or Claude) to produce an answer *grounded* in those documents

### Why RAG?

Without RAG, language models rely only on what they learned during training. This means they can:

- ❌ Hallucinate facts
- ❌ Miss recent events
- ❌ Lack domain-specific knowledge

With RAG, the model can:

- ✅ Access up-to-date information
- ✅ Cite sources
- ✅ Answer questions about *your* private documents

---

## How RAG Works

The RAG pipeline has three main stages:

### Stage 1 — Indexing (Offline)

```
Raw Documents → Chunking → Embedding → Vector Store
```

- Documents are split into smaller **chunks**
- Each chunk is converted into a **vector** (a list of numbers) using an embedding model
- Vectors are stored in a **vector database** (e.g., FAISS, Pinecone, Chroma)

### Stage 2 — Retrieval (At Query Time)

```
User Query → Embed Query → Similarity Search → Top-K Chunks
```

- The user's question is also embedded into a vector
- The system finds the **most similar chunks** from the database

### Stage 3 — Generation

```
[System Prompt + Retrieved Chunks + User Query] → LLM → Final Answer
```

- Retrieved chunks are passed as **context** to the language model
- The model generates a grounded, informed response

## NLP vs RAG – Key Differences

| Feature | NLP | RAG |
|--------|-----|-----|
| **Purpose** | Understand/process language | Retrieve + generate answers |
| **Knowledge source** | Training data only | External documents + training |
| **Updatable?** | No (needs retraining) | Yes (update the database) |
| **Typical use case** | Classification, tagging, parsing | Q&A, chatbots, search |

## Code Example

A simple tokenization example using Python and NLTK:

```python
import nltk
nltk.download('punkt')

from nltk.tokenize import word_tokenize, sent_tokenize

text = "NLP is fascinating. RAG makes language models more powerful."

# Sentence tokenization
sentences = sent_tokenize(text)
print("Sentences:", sentences)

# Word tokenization
words = word_tokenize(text)
print("Words:", words)
```

**Output:**
```
Sentences: ['NLP is fascinating.', 'RAG makes language models more powerful.']
Words: ['NLP', 'is', 'fascinating', '.', 'RAG', 'makes', 'language', 'models', 'more', 'powerful', '.']
```
